Octopress のエントリの投入方法

どうやるんだ、という事で自分用控えを作成。しかもこれが一発目のエントリだったり。ちなみに以下エントリを参照させて頂いております。

  • OctopressでGitHub無料ブログ構築。sourceをBitbucket管理。簡単ガイド！

まず以下でファイルを作成する模様。

$ bundle exec rake new_post['post title']
mkdir -p source/_posts
Creating new post: source/_posts/2013-11-02-post-title.markdown

む、タイトルがファイル名になっておりますね。適切なソレを指定する必要あり。デフォルトの状態は以下な模様。

---
layout: post
title: "post title"
date: 2013-11-02 11:22
comments: true
categories: 
---

念のため文字コードは UTF-8 にしておいて preview は以下で確認できる模様。

$ bundle exec rake preview

で、エントリ投入は

$ bundle exec rake gen_deploy

とありますが git な操作は不要なのかな。このエントリで試してみます。とりあえず保存のみしておいて上の二つを実行。

確認してみた

どうやら title を指定すると h1 な見出しは不要みたいですね。

む

接続失敗。

ssh: connect to host domain.com port 22: Connection refused

って _config.yml にはそれ風な記述が無い。どうやら Rakefile らしい。

もう少し

エントリ投入でちょっと苦労。以下を確認させて頂きつつ

  • Switching to Octopress

設定を初期化 (?) して

$ bundle exec rake setup_github_pages

デプロイして

$ bundle exec rake deploy

あと、昔作った残骸のようなものが yamanetoshi.github.com に存在してたのでリポジトリを削除して確認することができました。 エントリ修正して再度投入ができるのかどうか。
