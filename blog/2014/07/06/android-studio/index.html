Android Studio 久々に使ってみるなど

導入したらバージョンが 0.8.0 とか。手元の端末って当分更新してないんですがいくつなんだろ。

そりゃ良いとして導入して既存プロジェクトを import してみました。どうなるか。

ネイティブなライブラリ付きのソレだったんですが、import は問題なく完了。build に失敗してるみたいなので端末で gradlew build してみると SDK が未導入とのこと。これって既存なソレを参照とかってどうやるんだったか、と言いつつ SDK Manager を起動して導入。

終わったみたいなので build リトライしてみるに無事正常終了なのかどうか。

実行できない

実行構成を追加しなければ、だった模様。ただ、追加してもまだ駄目で SDK なコンポーネントが何か足りていなかったようです。あと、build.gradle の

apply plugin: 'android'

は

apply plugin: 'com.android.application'

て書けって言われました。

ハマりましたが

なんとなく以前より楽になってる気はしました。もう今後は Android Studio 使う方式でいいでしょ的な結論でどうか。あと手元の端末のバージョンは 0.4 系でした。

ちなみに SDK の参照先の変更は File -> Project Structure でした。

とは言え

実行構成の追加でハマッた程度で、後は指摘されてる事項についてリンクを踏むだけで何とかなりましたね。すばらです。
