RHG を読むことになった件

リモートで読書会、とか言ってるけど本当に成立するのだろうか。読むのは以下から手繰れる HTML 版なコンテンツです。

  • 『Rubyソースコード完全解説』サポートページ

とりあえず

GitHub 方面に Organization 作成などを、と思ったのですがギー沖な Organization に作成してとりあえず plusjade/jekyll-bootstrap を clone しつつ、作成したリポジトリもアレ。

違う。とりあえず

  • git remote add upstream https://github.com/plusjade/jekyll-bootstrap
  • git fetch upstream
  • git merge upstream/master

して push すれば良いのかな。

結局

ギー沖の Organization で GitHub Pages 作るのは申し訳ないので結局 Organization 作成してそっちに push しちゃいました。

読書会な手順

以下な順なのかどうか。

  • GitHub の Issues でやることを宣言する
  • 作成した Issues 番号と自分のIDを元にブランチを作成する
  • /_posts/yyyy-mm-dd-event-xxx.markdown に「自分の名前」と「作成したIssueへのリンク」を作成する
  • 作成したファイルに読んだログを控える
  • Pull Request 発行

Issue とかで色々掘り下げることができれば良いですね。
