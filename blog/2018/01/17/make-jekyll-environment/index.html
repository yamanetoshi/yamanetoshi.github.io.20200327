jekyll な環境構築

とりあえず作成してみることに。手順としては

  • ruby と bundler と jekyll 導入
  • jekyll new でプロジェクト作る
  • github-pages は Gemfile で云々

なカンジで。

とりあえず

ruby のバージョンは 2.4.2 で github-pages は 172 という方向。2.4.2 な ruby は導入済みでした。そして jekyll も 3.6.2 でした。この最近、酔っ払って云々したのかな。

プロジェクト作成

します。

$ jekyll new hogehoge

で、いっかい Gemfile* を削除して以下?

source "https://rubygems.org"

gem "github-pages", "172"
gem "rake"

で、以下。

$ bundle install --path vendor/bundle

出力ロストしましたがおそらく大丈夫なのではないかと。

プレビュー

出ました。Rakefile はてきとーな所からパクりました。とりあえずこれをプロトタイプ、ってことにして Github に push します。

追記

octopress な環境に影響出てたorz とりあえず octopress の最新に Gemfile を追従させております。

一旦

止めます。これは対応が大変そう。

その後

一旦、Gemfile あたりとか _config.yaml などを巻き戻して以下。

$ rbenv local 2.2.4

どうも別方面で使ってた 2.4.2 な ruby がアレで yajl あたりに影響が生じていたようです。
