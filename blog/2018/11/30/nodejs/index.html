Node.js

使って云々してるんですが、微妙な挙動でくっそハマッたので控えを (後々役に立つのかどうかわからない程度なナニです)。

対向? のサーバと REST な接続および WebSocket な接続を、っていう微妙な実装だったのですが対向が neo4j サーバでしてこれのフロントにしてる Gremlin サーバが両方をいっぺんに喋ることができない、というのが前提です。

  • 最初に REST な実装を盛り込んだ (勿論 REST な実装で試験)
  • 次に WS な実装を盛り込んだ (勿論 WS な実装で試験)

その後、再び REST な実装を盛り込んで試験をしようとしたところ、それまで正常に動いてたサービスが起動しなくなってしまったのです。対向なサービスが WS を喋らなくなったことが原因だったのですが、それを示すメッセージ的なものは一切出ず。意味のわからない試行錯誤の後 (2h 程度消費)、ようやく原因に辿りついた次第です。

それ以外にも

データ型マターな微妙な挙動を確認してる次第なのですが、Javascript というか Node.js 大変だな。
