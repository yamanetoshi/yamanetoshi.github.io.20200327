onShowFileChooser

html の input type が file のソレにアプリ側から画像を追加したいというソレ。微妙なハマリ方をしてしまったので自分メモを。

Duplicate showFileChooser result という例外を吐いていました。画像の Uri を確保して ValueCallback<Uri[]> filePathCallback の onReceiveValue に渡すあたり。ググッてみるに確かにソースコード的にも例外 throw している箇所は確認できたのですが二度も呼び出していない。

ググッても良さげな材料に行きあたらず、困り果てていたところ API リファレンスにて onShowFileChooser で true 戻せよな、って記述を発見してようやく解決。そういえば良く分からないまま、false を戻すようにしてました。

とは言え

まだキャンセル時の挙動がコントロールできてなくてハマり中はハマり中だったりしています。
