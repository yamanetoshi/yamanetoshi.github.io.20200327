Docker Registry

ようやく commit 作らない、が横着だった事に気づいた次第にて以下の方向でリポジトリを作成する事に。

  • S3 に情報保存
  • Dockerfile と config.yml を持って歩いて必要な時にコンテナ作って image 登録
  • pull する時にもその場でコンテナ立ち上げて云々する
  • 必要なファイル (Dockerfile, config.yml) は bitbucket 方面にて保管

ネト接続必須になりますが、これは気にする必要はないはず。

以下を参考にさせて頂きました。ありがとうございます。

  • docker-registry と Amazon S3 で作る自分だけのコンテナ倉庫 (リポジトリ)

動作確認してみた

動かんな、と思ってたらでびあん君に導入されてるのが以下でした。

$ sudo docker -v
Docker version 0.11.1, build fb99f99

1.6 を云々なのか、と言いつつ以下を確認。

  • Debian

とりあえず以下を実行。

$ sudo apt-get purge lxc-docker-0.11.1

で、apt-cache search してみたら lxc-docker-1.6.2 というソレが導入可能になっていました。と思ったら 301 が戻ってきているようなのでドキュメント通りにすすめます。

紆余曲折の末、1.10 が導入されたようです。

メモ

まだネト接続できておりません。ちなみにコンテナの IP は以下で確認できる模様。

$ sudo docker network inspect bridge
